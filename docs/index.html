> ea.js — Echelon Analytics

Privacy-first, self-hosted web analytics.
Drop in a single script tag — no cookie banners needed.

Why?

Google Analytics used to be my go-to. Then GA4 happened — bloated, confusing, and way too heavy. Using it feels like opening Microsoft Word when all you want is a text editor. So I built my own for afroute.com and ripped it out when implementing Islets Spatial CMS.

Quick Start

ECHELON_DB_PATH=./echelon.db deno run -A main.ts
<script src="https://your-host/ea.js" data-site="my-site"></script>

That's it. Pageviews, bounces, and sessions tracked automatically.

How It Works

  • 1x1 pixel beacon + sendBeacon for events
  • Cookieless by default — daily-rotating HMAC hash, no cross-day tracking
  • No cookie consent banners needed (GDPR/ePrivacy compliant)
  • Optional persistent cookies via data-cookie
  • SQLite with WAL mode — single file, zero config

Features

  • Auto-tracked pageviews, bounces, sessions
  • Click, scroll depth, and hover tracking
  • Bot detection with heuristic scoring (0-100)
  • A/B experiment tracking
  • Admin dashboard with realtime visitors
  • REST API for stats, bot management, experiments

Tech Stack

  • Deno + Fresh 2.2.0 (Preact)
  • SQLite via @db/sqlite
  • Preact islands + @preact/signals
